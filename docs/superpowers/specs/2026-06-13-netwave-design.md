# netwave — 终端 API 延迟电波图 设计文档

- 日期: 2026-06-13
- 状态: 已批准，待制定实现计划

## 一句话目标

在终端里用 Braille 示波器波形，实时监控本机到多个大模型 API 的 TCP 建连延迟，一眼看出哪家此刻最快/最慢、有没有丢包。

## 范围与关键决策

| 维度 | 决策 |
|------|------|
| 语言/运行 | Python 3.14，**零外部依赖**（仅标准库）。`python3 netwave.py` 直接启动，无需 venv/pip |
| 并发模型 | `asyncio`：每个目标一个探测协程 + 一个渲染循环 + 一个输入循环 |
| 终端输出 | 直接写 ANSI 转义序列（不使用 curses），面向 truecolor 终端（如 ghostty） |
| 测量方式 | 对 `host:port`（默认 443）做异步 TCP 建连计时，取连接建立耗时(ms)；不需要 API key、不产生计费调用 |
| 监控目标 | 多个大模型 API 端点，来自 TOML 配置文件（预置常见项 + 用户可增删） |
| 可视化 | Braille 示波器面板，每个 API 一个面板纵向堆叠，新样本从右进、向左滚动 |
| 采样间隔 | 默认 1.0s，可配置 |
| 超时表现 | 红色满格尖刺，并计入该面板 `loss%` |
| 颜色阈值 | 固定阈值（可配置）：`<100ms` 绿、`100–250ms` 黄、`>250ms` 红 |
| Y 轴缩放 | 每个面板按自身近期 max 自适应，底部 0 基线 |
| 交互 | `q`/`Ctrl-C` 退出（恢复终端），`p` 暂停/继续，`SIGWINCH` 重排 |

名称 `netwave` 为暂定名，可改。

## 模块划分

每个文件职责单一、边界清晰、可独立测试。

| 文件 | 职责 | 依赖 | 测试方式 |
|------|------|------|----------|
| `probe.py` | 对单个 `host:port` 做异步 TCP 建连计时，返回延迟(ms) 或超时标记 | asyncio | 用本地监听 socket 测建连成功与超时 |
| `buffer.py` | 每个目标的环形采样缓冲 + 滚动统计（现值/avg/max/min/丢包率/抖动） | 无 | 纯单元测试 |
| `braille.py` | Braille 画布：给定数值序列 + 宽高 + 颜色阈值，渲染成彩色波形行 | 无 | 快照测试输出字符串 |
| `render.py` | 组合面板（表头 + 波形 + 基线），按终端尺寸自适应排版 | braille, buffer | 快照测试 |
| `config.py` | 读取/解析 TOML 配置，提供默认值，首次运行自动生成默认配置 | tomllib | 解析 + 默认值 + 首次生成测试 |
| `app.py` | asyncio 主循环：调度探测、更新缓冲、定时渲染、键盘/resize/退出 | 上述全部 | 最小集成测试 |

## 数据流

```
config.toml ──► [targets] ──► 每个目标一个探测协程(每 interval 秒)
                                      │
                              probe → 延迟(ms) / 超时
                                      │
                                      ▼
                              buffer 环形缓冲 + 滚动统计
                                      │
渲染循环(每 interval 秒) ──► render 读缓冲 ──► braille 画波形 ──► 写一帧到终端
                                      ▲
                          输入任务(q 退出 / p 暂停) · SIGWINCH(resize)
```

## 屏幕布局

每个 API 一个面板，纵向堆叠，按可用终端高度平分。单面板示意：

```
api.anthropic.com          42ms   avg 48   max 120   loss 0%
120┤                              ⠙⢠
   │        ⢠           ⢠⡀        ⢰⠙⢤    ⢠⡀
 60┤   ⢠⣤⠒⠐⠉⠲⢤⡀  ⢠⣤⢰⠒⠐  ⠉⠲⢤⣀⠤⠐    ⠘⢢⡀⢠⠐ ⠉
  0┼──────────────────────────────────────────  ◀ 滚动
```

- 表头：API 名 + 现值 / avg / max / 丢包率，颜色随现值变化
- Braille 画布水平方向 2 点/字符列，可容纳「面板宽度 ×2」个采样点
- Y 轴每面板按自身近期 max 自适应缩放，底部为 0 基线
- API 数量多到放不下：先自动压缩单面板高度（设最小高度）；再不够则显示「终端太小」提示

## 颜色与超时

- 固定阈值（配置可改）：`<green` 绿、`green–yellow` 黄、`>yellow` 红（默认 green=100, yellow=250，单位 ms）
- 每个 Braille 字符按它所代表的延迟值着色（同一列取代表值，如该列的较大样本）
- 超时 / 建连失败 → 红色满格尖刺，并计入该面板 `loss%`

## 配置文件

TOML 格式，用标准库 `tomllib` 读取（只读解析；用户手工编辑）。

查找顺序：`--config 指定` → `./config.toml` → `~/.config/netwave/config.toml`。
都不存在时，在 `~/.config/netwave/config.toml` 生成带预置项的默认配置并使用之。

示例：

```toml
interval = 1.0                 # 采样间隔(秒)
timeout  = 2.0                 # 建连超时(秒)

[thresholds]
green  = 100                   # ms，低于此为绿
yellow = 250                   # ms，介于 green~yellow 为黄，高于为红

[[targets]]
name = "anthropic"
host = "api.anthropic.com"
port = 443

[[targets]]
name = "openai"
host = "api.openai.com"
port = 443
```

预置 targets：Anthropic / OpenAI / Google / DeepSeek 等常见大模型 API（用户后续随意增删）。

## 交互与健壮性

- `q` 或 `Ctrl-C` 退出；退出时恢复终端状态（显示光标、清屏、复位）
- `p` 暂停 / 继续渲染与探测
- 监听 `SIGWINCH`，终端尺寸变化时重新排版
- DNS 失败 / 连接被拒 / 超时 都记为一次丢失（红尖刺），程序绝不崩溃
- 配置缺失自动生成默认；配置损坏给出清晰错误信息后退出
- 终端过小（放不下任何完整面板）时显示提示而非崩溃

## 测试策略（TDD）

实现先写测试：

- `probe`：起一个本地监听 socket 验证建连计时；用不可达端口/极短超时验证超时分支
- `buffer`：滚动统计为纯函数，覆盖 avg/max/min/丢包率/抖动 与环形覆盖边界
- `braille`：固定输入下对输出字符串做快照断言，覆盖空数据、超时尖刺、颜色阈值分段
- `config`：解析合法/缺省/损坏配置，验证默认值与首次自动生成
- `app`：最小集成测试，跑通「探测→缓冲→渲染一帧」不崩溃

## 明确不做（YAGNI）

- 不做真实 API 请求计时（不需要 key、不计费）——只测 TCP 建连
- 不做历史持久化 / 导出 / 告警通知（首版聚焦实时可视化）
- 不做鼠标交互 / 多页签 / 配置热重载
